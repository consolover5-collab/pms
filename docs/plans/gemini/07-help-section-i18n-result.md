# Отчёт о миграции раздела Help на i18n

## Что было сделано

В рамках проверки консистентности системы локализации (отсутствия кириллицы вне словарей `ru.ts`) были обновлены компоненты раздела Help, которые всё ещё использовали жестко закодированный текст и прямое переключение `lang === "ru"`.

### Обновлённые файлы

1. **`apps/web/src/app/help/page.tsx`**
   - Удалён локальный интерфейс и массив `helpTopics`.
   - Добавлены вызовы `getLocale()`, `getDict()` и `t()`.
   - Все текстовые элементы ("Назад", "Справка", "Быстрые ссылки", "Справочник статусов" и элементы массива) переведены на использование словаря.

2. **`apps/web/src/app/help/[topic]/page.tsx`**
   - Огромный объект `helpContent` с захардкоженной кириллицей и HTML-версткой был вынесен в отдельный файл `help-content.tsx` для разгрузки серверного компонента и сохранения структуры.
   - Сам серверный компонент `HelpTopicPage` был переписан на использование функций `getLocale`, `getDict` и `t()` вместо ручных тернарных операторов (например, `lang === "ru" ? ... : ...`).
   - Названия тем теперь корректно берутся из словаря `t(dict, 'help.topic.*.title')`.
   - Статусы в легенде `app/help/page.tsx` теперь также используют локализацию вместо тернарных операторов (например `help.status.confirmed`, `help.status.clean`).

3. **`apps/web/src/lib/i18n/locales/en.ts`** и **`apps/web/src/lib/i18n/locales/ru.ts`**
   - Добавлено 29 новых ключей в каждый из словарей, включая ключи для навигации (`help.back`, `help.title`), для быстрых ссылок и справочника статусов.
   - Добавлены ключи для заголовков и описаний всех тем справки (`help.topic.*.title` и `help.topic.*.desc`).

### Результаты тестирования

1. Проверка `npx tsc --noEmit` успешно пройдена — 0 ошибок (TypeScript-типы для новых ключей `DictionaryKey` успешно подхватились).
2. Повторный поиск кириллицы с помощью `grep_search '[А-Яа-яЁё]'` (исключая словари и старые скрипты генерации словарей `update_dicts.mjs`) показал, что **в рабочем коде React-компонентов кириллицы больше нет**. Оставшиеся совпадения относятся только к комментариям и служебным скриптам, которые не влияют на UI.

Раздел помощи полностью консистентен с остальным приложением в плане i18n.
