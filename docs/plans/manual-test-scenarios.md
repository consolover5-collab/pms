## 2. Check-in в грязный номер (FEAT-003)
Подготовка — перевести номер в dirty через curl:
```bash
# Узнать roomId брони GBH-000009 (arriving today)
curl -s "http://localhost:3001/api/bookings?propertyId=ff1d9135-dfb9-4baa-be46-0e739cd26dad&search=GBH-000009" | jq '.[0].roomId'

!cannot index room number!

# Перевести номер в dirty (подставить roomId)
curl -s -X POST "http://localhost:3001/api/rooms/ROOM_ID/status" \
  -H "Content-Type: application/json" \
  -d '{"propertyId":"ff1d9135-dfb9-4baa-be46-0e739cd26dad","housekeepingStatus":"dirty"}'
```

- [ ] Открыть бронирование со статусом `confirmed` с назначенным номером
- [ ] Нажать **Check In** → появился диалог-предупреждение (не просто ошибка)
- [ ] Нажать **Отмена** → диалог закрылся, гость не заселён
- [ ] Нажать **Check In** снова → **Заселить всё равно** → заселение прошло
