# Промт для GLM5.1: Отладка — вкладка Folio не отображается

## Контекст
Проект: PMS на Next.js 16 (App Router) + React 19 + Fastify 5.
Монорепо: `apps/web/` (фронтенд), `apps/api/` (бэкенд, порт 3001).

## Проблема
На странице бронирования `/bookings/[id]` не отображается секция Folio (или вкладка Folio пустая).
Тестовое бронирование: **GBH-000003** (гость с привязанной компанией "Балтийские Линии", должно быть 2 окна фолио).

Ожидалось: вкладка/секция Folio с двумя windows (Window 1 — гость, Window 2 — компания).
Наблюдается: секция полностью отсутствует / пустая.

## Задача для исследования
1. Изучи `apps/web/src/app/bookings/[id]/` — найди где и как рендерится Folio.
2. Проверь API-эндпоинт: `GET /api/bookings/:id/folio` или аналогичный — что возвращает для GBH-000003.
3. Найди в `apps/api/src/routes/folio.ts` (или аналогичном файле) логику получения folio windows.
4. Проверь схему БД: таблицы `folio_windows`, `folio_transactions` или аналогичные в `packages/db/src/schema/`.
5. Выяви причину: либо API возвращает пустые данные, либо компонент не рендерит при непустых данных, либо ошибка в условии отображения.

## Порядок действий

1. Проведи исследование (пункты 1–5 выше) — найди причину.
2. Сделай код-ревью затронутых файлов: нет ли рядом других проблем того же рода.
3. **Примени фикс напрямую в файлах** — не описывай, а измени код.
4. Сохрани отчёт в `docs/plans/glm/02-folio-tab-missing-result.md`:
   - Диагноз: где обрывалась цепочка (API / компонент / условие), file:line
   - Что изменено (краткий diff или список файлов и строк)
   - Как проверить: curl-запрос + действие в UI
