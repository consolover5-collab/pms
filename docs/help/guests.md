# Гости (Guests)

Управление профилями гостей - хранение контактной информации и истории проживания.

## Расположение
- URL: `/guests`
- Создание: `/guests/new`
- Просмотр: `/guests/[id]`
- Редактирование: `/guests/[id]/edit`

## Информация о госте

### Основные данные

| Поле | Описание | Обязательное |
|------|----------|--------------|
| **First Name** | Имя | ✅ Да |
| **Last Name** | Фамилия | ✅ Да |
| **Email** | Электронная почта | Нет |
| **Phone** | Телефон | Нет |

### Дополнительные данные

| Поле | Описание |
|------|----------|
| **Date of Birth** | Дата рождения |
| **Nationality** | Гражданство (код страны) |
| **Document Type** | Тип документа (passport, id_card) |
| **Document Number** | Номер документа |
| **Address** | Адрес |
| **City** | Город |
| **Country** | Страна |
| **Postal Code** | Почтовый индекс |
| **Notes** | Примечания |
| **VIP Level** | Уровень VIP (0-5) |

## Создание гостя

### При создании бронирования

1. В форме создания бронирования нажмите **New Guest**
2. Заполните данные
3. Гость автоматически привяжется к бронированию

### Отдельно

1. Перейдите в **Guests**
2. Нажмите **+ New Guest**
3. Заполните минимум: Имя и Фамилию
4. Нажмите **Create Guest**

## Просмотр профиля

Карточка гостя показывает:
- Контактную информацию
- Данные документа
- Адрес
- VIP уровень
- Примечания

## Редактирование

1. Откройте профиль гостя
2. Нажмите **Edit**
3. Измените нужные поля
4. Нажмите **Save**

## Поиск гостей

В списке гостей можно искать по:
- Имени
- Фамилии
- Email
- Телефону

## VIP уровни

| Уровень | Описание |
|---------|----------|
| 0 | Обычный гость |
| 1 | Постоянный гость |
| 2 | Важный гость |
| 3 | VIP |
| 4 | VIP+ |
| 5 | VVIP |

VIP уровень используется для:
- Приоритетного обслуживания
- Назначения лучших номеров
- Особых условий

## Типы документов

| Код | Описание |
|-----|----------|
| `passport` | Паспорт |
| `id_card` | Удостоверение личности |
| `drivers_license` | Водительские права |
| `other` | Другой документ |

## Связь с бронированиями

Один гость может иметь несколько бронирований. При просмотре профиля видна история бронирований гостя.

## Дубликаты

При создании нового гостя проверяйте, нет ли уже такого профиля в системе. Поиск по имени/email поможет избежать дубликатов.

## Защита данных

Персональные данные гостей должны обрабатываться в соответствии с законодательством о защите персональных данных:
- Храните только необходимые данные
- Не передавайте данные третьим лицам
- Удаляйте данные по запросу гостя
